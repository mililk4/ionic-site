Contribute To Ionic

Help build awesomeness! If you see an area for improvement, or have an idea for a new feature then we'd love to have your help! Additionally, please feel free to take part in any discussions which may be happening at our issues page.

To get started contributing to Ionic:

  1. Browse existing issues for ideas and take part in any discussions to help define what you want to do.
  2. Fork the Github repo.
  3. Create a new branch from master: git checkout -b issuenumber-new-awesomeness
  4. Develop your feature.
  5. Follow our coding conventions.
  6. Did you remember to write a passing test?
  7. Submit a pull request!

For issues relating to the Ionic documentation or this site, use the ionic-site repo instead.

Submit An Issue On Github

If you catch a typo, or see a way to improve any piece of code please let us know. There's no such thing as too small of an improvement.

Conventions

We try to follow best practices as much as possible.

For any SCSS changes, try to keep CSS properties sorted in a reasonable order, with layout and sizing properties first, then cosmetic properties below that. See our code for examples. Feel free to run your code through CSS Comb, which will sort your CSS properties in a good order.

All JavaScript must pass JSHint, which is helps detect errors and potential problems in your code.

Additionally, almost all Javascript code must have an accompanying test case. To run our test suite, install and run Karma as follows: 

$ sudo npm install -g karma
$ karma start ionic.conf.js

Any tests you write should go in the test/js folder for core JS code, or in the js/ext/FRAMEWORK/test folder for framework-specific code. Copy the format from other tests to write your own.

Indentation

Always use 2 spaces with no tabs.
