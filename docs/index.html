Utility Classes

Many components in Ionic purposely have both padding and margin reset set to zero. In many instances mobile apps will have components bleed to the edge of the screen, and by starting each component at zero, developers can easily control padding throughout the app.

padding
The padding class it used to give any element's content some breathing room, meaning it adds padding between the edge of the element and its inner content.

Buttons

Buttons can be displayed inline, block, outlined, icon-only, or clear (text only), and come in many different styles. By default a button has display: inline-block applied. Other options include block buttons for a full width.

{% highlight html %} {% endhighlight %}

Block Buttons

Adding block to a button applies display: block display. A block button will however go 100% of its parent's width. In the example, the button's containing content element also has padding applied, so there is some breathing room between the edge of the device and the buttons.

{% highlight html %} {% endhighlight %}
Block Buttons

Full Width Block Buttons

Adding button-full to a button not only applies display: block, but also removes borders on the left and right, and any border-radius which may be applied. This style is useful when th button should stretch across the entire width of the display. Additionally, the button's parent element does not have padding applied.

List Icons

Lists can have icons assigned either to the left and/or right side of each list item, and the alignment classes should be assigned to each list-item element.

Use list-icon-left to line up the icon to the left, and list-icon-right to set the icon to the right. When a list item has an icon on both sides then both classes will need to be applied.

In the example, the first list item only has a left aligned icon. The second item has both left and right side icons, and the third only has a right side icon. Additionally, the first icon has fill-icon applied so it inverts the background and foreground.

This example uses an <a> element for each item, which allows the entire list item to be tappable. Note that both a list and its list items can be any element, meaning they're not restricted to just <ul> with <li> elements.

Forms & Inputs

An input-group is used to group related input elements. An input-row is used to designate each individual input field and it's associated label. By default each input row will fill 100% of the width of its parent element. However, you can use the utility classnames of inset and padding to change their placement.

Ionic prefers to create an input-row out of the <label> element so that when any part of the row is tapped then the input receives focus.
