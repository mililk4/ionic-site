Layout & Structure

Ionic's layout is first based off the concept of an application, which is in contrast to a traditional web site. An application has fixed areas, and can scroll its content, whereas with a traditional web site the entire area is scrollable. To accomplish the web app effect within a web browser you'll need to follow certain HTML/CSS guidelines.

Layout Utility Classes

One common theme throughout Ionic is to give the developers more control of margins and padding the layout. Going this route allows the developer to explicitly state how the element fits within the layout, and doesn't require a mess of overrides to accomplish the layout your looking for. As a general rule, most elements throughout the framework are set to margin: 0; and padding: 0; which allows more explicit classes to be easily assigned.

inset
The inset class it used to "inset" any element, meaning it adds a margin around the element.
padded
The padded class it used to give the element's content some breathing room, meaning it adds padding between the edge of the element and the inner content.
