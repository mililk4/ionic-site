Buttons

Buttons can be displayed inline, block, icon-only, or text only, and come in many different styles.

{% highlight html %} {% endhighlight %}

Block Buttons

Block buttons take up the full width of their parent container, and make themselves really hard to miss. Add the button-block class to make any button a block button.

Text-only Buttons

Text-only buttons are a predominate feature of iOS 7. Add button-clear to any button to remove the border and make the text stand out. Note: The text will take the color of the applied button style. Meaning button-primary will result in blue text instead of a blue background.

Icon-only Buttons

While icons can be added to any button at any time, to remove any border and make the icon stand alone, add button-icon to the button. This also reduces the padding applied to the button.

Buttons in Headers/Footers

When buttons are placed in headers or footers, they take the style of the bar by default, so you don't have to use the extra style classes. To change this, add the desired style class.

Layout Utility Classes

Many components in Ionic do not have padding or margin by default, to make it easier to control the inset look of certain controls like lists and content areas.

inset
The inset class it used to "inset" any element, meaning it adds a margin around the element.
padded
The padded class it used to give the element's content some breathing room, meaning it adds padding between the edge of the element and the inner content.
