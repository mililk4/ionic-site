Content

The content area in Ionic is the scrollable viewport of your app. While your headers and footers will be fixed to the top and bottom, respectively, the content area will fill the remaining available space.

The content area also sits "behind" the header and footer to provide a see-through effect. Primary content element should have the content class applied. No thrills here other than default appearances for the various type element.

Note: The containing content element also has the padding class applied (more about padding is discussed a few sections later).

Utility Classes

Many components in Ionic purposely have both padding and margin reset set to zero. In many instances apps will have components bleed to the edge of the screen, and by starting each component at zero, developers can easily control padding throughout the app.

The padding utility classes can be reused to give any element's content some breathing room, meaning it adds a default 15px between the outer box of the element and its inner content. The following classes are not required for any element, but may be helpful with your designs.

  • padding Adds padding around every side.
  • padding-vertical Adds padding to the top and bottom.
  • padding-horizontal Adds padding to the left and right.
  • padding-top Adds padding to the top.
  • padding-right Adds padding to the right.
  • padding-bottom Adds padding to the bottom.
  • padding-left Adds padding to the left.

Ah, the Button, an essential part of any mobile experience. Buttons are commonly used to send/retrieve data and form navigation between screens. Like the Header, they come in the full spectrum of Ionic's default color options.

Buttons can be displayed inline, block, outlined, icon-only, or clear (text only), and come in many different styles. By default a button has display: inline-block applied. Other options include block buttons for a full width.

Block Buttons

Adding block to a button applies display: block display. A block button will however go 100% of its parent's width. In the example, the button's containing content element also has padding applied, so there is some breathing room between the edge of the device and the buttons.

Full Width Block Buttons

Adding button-full to a button not only applies display: block, but also removes borders on the left and right, and any border-radius which may be applied. This style is useful when the button should stretch across the entire width of the display. Additionally, the button's parent element does not have padding applied.

Cards are a relatively new visual design paradigm on mobile, though they've become widely used in recent years. And for good reason- cards are a great way to contain and organize information, while also setting up predictable expectations for the user. With so much content to display at once, and often so little screen realestate, cards have fast become the design pattern of choice for many companies, including the likes of Google, Twitter, and Spotify..

For mobile experiences, Cards make it easy to display the same information visually across many different screen sizes. They allow for more control, are flexible, and can even be animated. Cards are usually placed on top of one another, but they can also be used like a "page" and swiped between, left and right. Dive in and make some cool cards! {% highlight html %}

Popups

Moduleionic.service.popup
ServicePopup

Popups are "modal" popup windows like alert, confirm, and prompt popups used in many Javascript-powered websites. In a web-powered web or native app, alert(), confirm(), and prompt() can still be used, but to get more control over the experience, use the Popup service. Note, these popups are asynchronous unlike the core native versions, and will not suspend running application.

{% highlight javascript %} // Show an alert popup Popup.alert('Bad user!'); // Show a prompt popup Popup.prompt('I never asked, do you like Guacamole?', function(response) { }); // Show a confirm popup Popup.confirm('Are you sure you want to throw it all away?', function(response) { }); // Show a custom popup with two custom buttons Popup.show({ title: 'Custom popup', buttons: [ { 'title': 'Button 1', onClick: function() {} }, { 'title': 'Button 2', onClick: function() {} }, ] }); {% endhighlight %}
