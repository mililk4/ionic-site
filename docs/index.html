Utility Classes

Many components in Ionic purposely have both padding and margin reset set to zero. I many instances mobile apps let components bleed to the edge of the screen, and by starting each component at zero this allows developers to easily control the common margins and padding throughout the app.

margin
The margin classname is used to "inset" any element from its parent container. Meaning it adds a common margin between itself and its parent. Margin is useful when its parent has no margins at all, but a specific element should be inset a common distance.

For example, text inputs may want to stretch 100% of the display, however, the button to submit the form may need to be inset for a better apperance.

padding
The padding class it used to give the element's content some breathing room, meaning it adds padding between the edge of the element and its inner content.

Buttons

Buttons can be displayed inline, block, outlined, icon-only, or clear (text only), and come in many different styles. By default a button has display: inline-block applied. Other options include block buttons for a full width.

Block Buttons

Adding block to a button applies display: block display. A block button will however go 100% of its parent's width. In the example, the button's containing content element also has padding applied, so there is some breathing room between the edge of the device and the buttons.

Block Buttons

Full Width Block Buttons

Adding button-full to a button not only applies display: block, but also removes borders on the left and right, and any border-radius which may be applied. This style is useful when th button should stretch across the entire width of the display. Additionally, the button's parent element does not have padding applied.

Use button-outline to apply an outline button style, which also has a transparent background. Note: The text and border will take the color of the applied button style, meaning button-primary will result in blue text and border, with a transparent background.

Forms & Inputs

An inputs-group is used to group related input elements. An input-row is used to designate each individual input field and it's associated label. By default each input row will fill 100% of the width of its parent element. However, you can use the utility classnames of inset and padding to change their placement.

Ionic prefers to create an input-row out of the <label> element so that when any part of the row is tapped then the input receives focus.
