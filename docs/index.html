Footer

Utility Classes

Many components in Ionic purposely do not have padding or margin by default. By starting each component at zero, this allows developers to easily control the inset and layout look of any component.

inset
The inset class it used to "inset" any element, meaning it adds a margin around the element.
padded
The padded class it used to give the element's content some breathing room, meaning it adds padding between the edge of the element and the inner content.

Clear Buttons

Clear buttons are a predominate feature in modern apps. Add button-clear to any button to remove the border and make the text stand out. Note: The text will take the color of the applied button style. Meaning button-primary will result in blue text instead of a blue background. By default clear buttons have display: inline-block applied.

Forms & Inputs

An inputs-group is used to group related input elements. An input-row is used to designate each individual input field and it's associated label. By default each input row will fill 100% of the width of its parent element. However, you can use the utility classnames of inset and padded to change their placement.

Ionic prefers to create an input-row out of the <label> element so that when any part of the row is tapped then the input receives focus.
