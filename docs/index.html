Utility Classes

Many components in Ionic purposely have both padding and margin reset set to zero. I many instances mobile apps let components bleed to the edge of the screen, and by starting each component at zero this allows developers to easily control the common margins and padding throughout the app.

margin
The margin classname is used to "inset" any element from its parent container. Meaning it adds a common margin between itself and its parent. Margin is useful when its parent has no margins at all, but a specific element should be inset a common distance.

For example, text inputs may want to stretch 100% of the display, however, the button to submit the form may need to be inset for a better apperance.

padding
The padding class it used to give the element's content some breathing room, meaning it adds padding between the edge of the element and its inner content.

List View

The list view is a very versitile and powerful component used to display lists of data. List views support various interaction modes such as editing, swipe to edit, drag to reorder, and pull to refresh. List views can be used in virtual mode to efficiently handle large lists of data by only rendering the items the user can see.

{% highlight html %} {% endhighlight %}

Lists have many options that can be configured and events that can be handled:

Attribute Type Default Description
virtual boolean false Whether to virtually handle list items, offloading items that are not in the viewport. This option may have unintended side-effects for certain applications, so it is not enabled by default. However, it is recommended for large lists.
is-editing boolean false A boolean value indicating whether to put the list in "edit mode"
on-refresh function Called when the list has been pulled to refresh and is now refreshing.
on-refresh-opening function(ratio) A callback triggered as the list is being pulled to refresh. This triggers for every drag event and sends the ratio of how far down the list is being pulled before it refreshes (between 0 and 1).
on-refresh-holding function A callback triggered when the list is being pulled down and held before refreshing.
animation className An animation class to apply on the list for animating children.
delete-icon icon className icon-minus-circled The class name of the icon to show for all deletion items. We recommend the default ionicon icon-minus-circled
reorder-icon icon className icon-navicon The class name of the icon to show for reordering. The recommended default is icon-navicon

List Items

List items make up the bulk of lists. List items can be swiped, reordered, deleted, and selected. They support hidden button bars that are exposed after swiping, and trigger events on deletion and selection.

List items also support embedded form elements and right-hand icons.

The content specified in the item content area will be the main content of the displayed item. Make sure to add an icon at the end if you want a right-handed icon.

can-delete boolean Whether this item can be deleted by the user.
can-reorder boolean Whether this item can be reordered.
can-swipe boolean Wheter this item can be swiped to expose the hidden buttons.
on-delete function Called when an item is deleted by the user.
on-select function Called when an item is selected or "tapped" by the user.

Forms & Inputs

An inputs-group is used to group related input elements. An input-row is used to designate each individual input field and it's associated label. By default each input row will fill 100% of the width of its parent element. However, you can use the utility classnames of inset and padding to change their placement.

Ionic prefers to create an input-row out of the <label> element so that when any part of the row is tapped then the input receives focus.

Navigation Stack

Moduleionic.ui.nav
Directivenavs (element)

The Navigation Stack is a stack of screens the user has navigated to, and can travel back in history to. This API is modeled from the UINavigationController in iOS.

The Navigation Stack powers a Header Bar, including changing the active title of the bar, to hiding and showing a back button if available.

{% highlight html %} {% endhighlight %}

To interact with the Nav Controller from your own controllers or directives, access the $scope.navController object on prototypically-inherited child scopes.

To add new screens to the navigation controller, call $scope.navController.pushFromTemplate(templateUrl) to navigate to a new content area.

Popups

Moduleionic.service.popup
ServicePopup

Popups are "modal" popup windows like alert, confirm, and prompt popups used in many Javascript-powered websites. In a web-powered web or native app, alert(), confirm(), and prompt() can still be used, but to get more control over the experience, use the Popup service. Note, these popups are asynchronous unlike the core native versions, and will not suspend running application.

{% highlight javascript %} // Show an alert popup Popup.alert('Bad user!'); // Show a prompt popup Popup.prompt('I never asked, do you like Guacamole?', function(response) { }); // Show a confirm popup Popup.confirm('Are you sure you want to throw it all away?', function(response) { }); // Show a custom popup with two custom buttons Popup.show({ title: 'Custom popup', buttons: [ { 'title': 'Button 1', onClick: function() {} }, { 'title': 'Button 2', onClick: function() {} }, ] }); {% endhighlight %}
